# Dev Landing — лендинг-презентация разработчика

Небольшой одностраничник: рассказ о себе, стеке, проектах, плюс рабочая форма обратной связи с отправкой писем владельцу и копии пользователю, и AI-помощник, который оформляет черновик комментария.

Тестовое задание: показать полный цикл **frontend → API → обработка ошибок → результат**.

---

## Стек

**Frontend (без фреймворков, только то, что просили в задании):**
- TypeScript
- HTML5
- SCSS
- Vite (dev-server, сборка)

**Backend:**
- Node.js + Express
- Nodemailer (отправка писем по SMTP)
- express-rate-limit (защита от спама)
- dotenv

**AI-интеграция:**
- OpenRouter API (по умолчанию модель `openai/gpt-4o-mini`, можно поменять в `.env`)

## Как запустить

### 1. Установить зависимости

\`\`\`bash
npm install
\`\`\`

### 2. Настроить переменные окружения

Скопируйте `.env.example` в `.env` и заполните значения:

\`\`\`bash
cp .env.example .env
\`\`\`

Минимум, что нужно для писем (Nodemailer / SMTP):
\`\`\`
SMTP_HOST=smtp.gmail.com
SMTP_PORT=465
SMTP_SECURE=true
SMTP_USER=ваш_email@gmail.com
SMTP_PASS=app_password         # для Gmail — App Password, не обычный пароль
MAIL_FROM="Dev Landing <ваш_email@gmail.com>"
OWNER_EMAIL=ваш_email@gmail.com
\`\`\`

Для AI-помощника:
\`\`\`
OPENROUTER_API_KEY=sk-or-v1-...
OPENROUTER_MODEL=openai/gpt-4o-mini
\`\`\`

> Если SMTP не заполнить — фронт отработает корректно, но при отправке формы сервер вернёт понятную ошибку, что почта не настроена.
> Если не заполнить `OPENROUTER_API_KEY` — AI-кнопка вернёт ошибку, основная форма продолжит работать.

### 3. Dev-режим

В двух процессах одновременно — Vite (фронт) и Node (бэк):

\`\`\`bash
npm run dev
\`\`\`

- Фронт: http://localhost:5173
- Бэк: http://localhost:3001
- Все запросы `/api/*` с фронта проксируются на бэк (см. `vite.config.ts`).

### 4. Прод-сборка

\`\`\`bash
npm run build       # собирает фронт в ./dist
npm run start:prod  # запускает Node.js, который раздаёт ./dist и /api/*
\`\`\`

В проде один процесс отдаёт и статику фронта, и API — удобно для деплоя на любой VPS / Render / Railway / Fly.io.

## Как реализована форма

**Поля:** имя, телефон, email, комментарий.

### Frontend (`src/contact-form.ts`)
1. Перед отправкой — клиентская валидация (`src/validation.ts`): длины, регулярки на email и телефон.
2. Состояния кнопки и статуса:
   - `idle` — обычное состояние;
   - `loading` — спиннер в кнопке, кнопка disabled, статус «Отправляем…»;
   - `success` — зелёное сообщение «Готово! Я получил заявку, копия письма ушла вам на email»;
   - `error` — красное сообщение с понятным текстом (валидация, сеть, ошибка сервера).
3. При вводе ошибка у поля сбрасывается автоматически.
4. Запрос: `POST /api/contact` с JSON-телом.

### Backend (`server/index.js`)
1. Лимит запросов — 5 в минуту с одного IP (`express-rate-limit`).
2. Валидация на сервере (`server/utils.js`) — независимо от клиента.
3. При успехе отправляются **два письма параллельно** через `nodemailer`:
   - **Владельцу сайта** (`OWNER_EMAIL`) — заявка с данными, в `Reply-To` — email пользователя, чтобы можно было ответить одним кликом.
   - **Копия пользователю** на его email — подтверждение получения.
4. Обработка ошибок:
   - 400 + `errors` — если валидация не прошла (фронт подсветит конкретные поля);
   - 500 / 502 — если SMTP не настроен или почта упала;
   - если копия пользователю не ушла, а владельцу ушла — заявка считается успешной, в ответе `userCopySent: false`.

## AI-интеграция

**Что:** на форме есть кнопка «✨ AI: помочь с текстом». Пользователь пишет в textarea 1–2 фразы (например, «нужен лендинг для крипто-стартапа, бюджет небольшой») и нажимает кнопку — ИИ переписывает это в нормальный, короткий комментарий от лица пользователя.

**Как:**
- Фронт: `POST /api/ai/draft` с `{ idea }`.
- Бэк: проксирует запрос в **OpenRouter** (`/api/v1/chat/completions`), с system-промптом, который заставляет модель:
  - писать на русском,
  - 2–4 предложения, по делу,
  - без приветствий и подписей,
  - не выдумывать фактов сверх идеи пользователя.
- Лимит: 10 AI-запросов в минуту с IP.
- Все ошибки (нет ключа, ошибка модели, пустой ответ) — обработаны и возвращаются понятным текстом.

Зачем именно OpenRouter, а не OpenAI напрямую: один ключ → доступ ко всем моделям (OpenAI, Anthropic, Google и т.д.), легко переключаться через `OPENROUTER_MODEL` без изменения кода.

## Какие AI-инструменты использовались при разработке

Около **95% кода** этого проекта написано с помощью ИИ — это мой обычный рабочий процесс.

**Что использовалось:**
- **v0 / Claude / GPT** — генерация бойлерплейта, верстки, валидации, серверных эндпоинтов, шаблонов писем, README.
- **Cursor / встроенные ассистенты** — рефакторинг и правки по месту.
- **OpenRouter** — для самой AI-фичи в продукте (черновик комментария).

**Что я делал руками:**
- выбор технологий для проекта
- структура проекта
- выбор темы UI/UX
- системный промпт для ии помошника в форме
- проверка работосопсобности всех вич проекта
- некоторые исправления в текстах и стилях

**Подход в целом:** ИИ — это инструмент для скорости. Если он быстрее меня собирает рабочий каркас — отлично. Но архитектурные решения, нестандартная бизнес-логика, интеграции и UX-нюансы — это всегда я, потому что ИИ без контекста проекта в этих местах регулярно ошибается.

В моих собственных продуктах я регулярно делаю похожие вещи: AI-анализаторы для рутины, ботов-ассистентов, мульти-агентные сценарии с RAG, AI-помощников в торговых инструментах. Так что это тестовое — маленький срез того, как я обычно работаю.

Есть базовые заготовки для использования в любом проекте промптов, что бы не делать одну и ту же работу по оптимизации много раз, так как проектов не мало. Это правила, материалы, ассеты может быть, которые я использую в своих проектах.
